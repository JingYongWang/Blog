CSS-Module

一開始看到這篇文章的標題不僅僅是好奇,
看完之後它告訴我了兩個重點:

  1. 不用想變數命名XD
    => 使用composes,指載入外部CSS,命名使用hash方式不會重複
  2. 效能提升 Why??
    => 原本要找整個頁面的某個標籤,現在只須找這個頁面的”唯一變數”!!

看完這篇覺得CSS-Module在前端部分以後可以成為程式規範之一
由於自己還未接觸React webpack等,
所以先筆記下來

主要設定寫在webpack.config.js檔,
網站內容是寫在 webpack-demo\src\components ,有引用ejs模板,分別把每段內容分成一個資料夾
其中資料夾裡的CSS可以看到都用相同的class name,
每段XXXDemo.js經過imports的CSS在設定檔才會被css-loader轉成hash方式的命名
