JS_Understanding_the_Weird_Parts-Part1

Note for JS Understanding the Weird Parts
克服JS的奇怪部分

名詞介紹與基本概念

  • Syntax parsers(語意分析器)

    把程式碼翻譯成讓電腦讀懂的指令,類似compiler 或 interpreter (直譯器)

  • Execution context 執行環境

    目前程式執行的環境(全域，或是在哪個function，或變數)

  • Lexical environment 詞彙環境 -code執行到哪邊

  • Javascript在執行時會先在記憶體空間擺入變數 與 函數 (關鍵字Hoisting)

    如果先呼叫函數，還是會執行，但變數未定義前呼叫則是 Undefined

  • 變數一開始的值為 undefined

  • undefined != not defined

    undefined : 未定義 => 有變數但沒定義值或形態
    not defined : 尚未定義 => 沒有定義這個變數

  • singel threaded 單執行緒

    一個執行一個指令(JS從程式碼角度看)

  • synchronous execution 同步執行

    一次一個

  • FUNCTION invocation 呼叫函數

  • Scope Chain 範圍鏈

  • 範圍代表能取變數的地方

  • 外部環境參照的連結

  • asynchronous 非同步 一個時機點不只一個
  • Dynamic Type JS 動態型別(處理型別的方式)

  • Static Type

    EX:Java使用靜態型別，代表一開始告訴編譯器你的變數是甚麼型別

  • JS 有六種Primitive type 基本型別

    1.undefined 表示還不存在(未設變數給它)
    2.Null 代表不存在，空值(可以設變數給它)
    3.Boolean ture or false
    4.number(可能是float或整數)
    5.string 字串
    6.symbol (for es6)

  • 運算子 operators 特殊的function
  • Precedence 優先性 Associativity 相依性
  • coercion 強制型轉
  • object Literals 物件實體
  • Faking NameSpaces 偽命名空間

變數與物件建立觀念
